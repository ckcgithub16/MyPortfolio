Checkers Game


This project was a collaboration with a classmate on the final project for an introductory Java computer science course. We created a basic checkers game that allows players to move around and compete with each other live (and on the same computer as the program is being run). The game will terminate when 1) one player either "kills" all of the other player's pieces by jumping over them with his own or 2) a player moves to the same place three times, meaning that the side with the most pieces in the end wins (in this situation). A banner is dislpayed across the tiles declaring that either black or white has won when either criteria is met. The gameplay also allows for "promotion" of a normal piece to a "king" when said piece has moved to the end of the opposing team's side. This king can move one space diagonally (or two spaces if jumping over an enemy) in the north and south direction, while normal pieces can only move in one direction before they're promoted.
